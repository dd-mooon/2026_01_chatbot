## 01. 프로젝트 개요 (Introduction)

### 배경

- 신규 입사자의 반복적인 기초 정보(위치, 규정) 문의로 인한 기존 인력의 업무 몰입 저해
- 사내 가이드 문서의 파편화로 정보 검색 시간 낭비

### 목적

- "물어보기 미안한" 사소한 질문의 창구 단일화
- 온보딩 프로세스의 자동화 및 효율화

### 기대 효과

- 신규 입사자 적응 시간 단축
- HR/총무팀의 반복 문의 응대 시간 절감

---

## 02. 서비스 벤치마킹 (무신사 '촤비스')

### 레퍼런스 분석

- **친숙한 메신저 UI:** 슬랙/카톡 스타일로 접근성 극대화
- **하이브리드 답변:** 정확한 정보는 DB에서, 복합 질문은 AI가 처리

### 본 프로젝트의 차별점

- **퍼블리셔 강점 활용:** React 활용한 더 유연하고 빠른 UI 인터랙션
- **실시간 고도화:** 어드민에서 즉시 지식을 보강하는 '데이터 피드백 루프' 구현

---

## 03. 상세 기능 기획 (Main Features)

### 03-1. 챗봇 (User)

**하이브리드 응답 로직**

- **1단계(키워드):** "건전지", "주차" 등 단어 매칭 시 정확한 위치/절차 안내
- **2단계(AI):** 키워드 부재 시 Ollama (Local LLM) API가 사내 문서를 기반으로 답변 생성

**UI 포인트**

- 답변 하단 관련 사내 링크(사내 게이트) 자동 연동
- 자주 묻는 질문 칩(Chip) 버튼 제공

### 03-2. 어드민 (Admin)

**지식 베이스(Knowledge Base) 관리**

- 키워드–답변–링크 셋을 관리자가 직접 등록/수정 가능한 대시보드

**미응답 질문 수집 및 고도화**

- 챗봇이 답변하지 못한 질문을 리스트업하여 '미답변 질문'으로 분류
- 관리자가 해당 질문에 대한 답을 등록하면 즉시 챗봇 지능 업데이트

### 03-3. 상세 기능 정리

| 구분 | 내용 | 효과 |
|------|------|------|
| **실시간 미응답 수집 (Logging)** | 챗봇이 답변하지 못한 질문을 자동 감지 및 DB 저장 | 임직원이 궁금해하는 '정보의 공백'을 실시간 데이터로 파악 |
| **자율적 지식 관리 (Admin)** | 관리자 UI를 통한 지식 등록·수정·삭제 (No-Code 운영) | 개발자 개입 없이 사내 변화(규정, 위치 등) 즉시 반영 |
| **실시간 지능 업데이트 (Real-time)** | 등록된 지식을 별도 서버 재시작 없이 즉시 서비스 적용 | 사용자가 물어볼수록 똑똑해지는 선순환 서비스 구조 구축 |

## 04. 기술 스택 및 시스템 구조

### 기술 스택

| 영역 | 기술 |
|------|------|
| **Frontend** | React.js (반응형 웹 및 애니메이션) |
| **Backend** | Node.js (Express) — Ollama API 중계 및 로직 처리 |
| **Database** | ChromaDB — 로컬 기반의 가벼운 벡터 DB |
| **AI Engine** | Ollama (Local LLM) — 오픈 API 연동 및 테스트 |

### 시스템 구조도

```
[User] ↔ [React UI] ↔ [Node.js Backend] ↔ [ChromaDB / Ollama]
```

## 05. 추진 일정 및 향후 계획 (Roadmap)

### 1단계: 기획 및 백엔드 인프라 구축 (1~2주차)

- **선작업:** Node.js 서버 환경 구성 및 Ollama 로컬 모델 연동 테스트
- **ChromaDB 세팅:** 사내 지식 데이터를 벡터화하여 ChromaDB에 적재하는 파이프라인 구축
- **협업:** 디자인팀 전소연 과장님과 UI/UX 컨셉 확정

### 2단계: 프론트엔드 퍼블리싱 및 API 연동 (3~5주차)

- **주요 작업:** React + Tailwind CSS 기반의 채팅 및 어드민 UI 구현
- **백엔드 연결:** Node.js 서버와 React 프론트엔드 간의 API 통신 구현

### 3단계: AI 연동 및 RAG 로직 구현 (6~7주차)

- **주요 작업**
  - **RAG 구현:** ChromaDB 검색 결과와 Ollama 모델을 결합한 답변 생성 로직 최적화
  - **로직 최적화:** [키워드 매칭 → ChromaDB 검색 → AI 답변] 순의 프로세스 고도화

### 4단계: 안정화 및 최종 배포 (8주차)

- **주요 작업**
  - 전체 시나리오 테스트 및 UI 디테일(마이크로 인터랙션) 폴리싱
  - Vercel을 활용한 실시간 웹 배포 및 최종 점검
  - 사내 시연을 위한 가이드 문서 및 발표 자료 완성

### 향후 계획

- 사내 메신저 연동 및 확장 가능성 검토
