# 코니(Connie) 프로젝트 

## 1. 이 프로젝트가 하는 일 (한 줄 요약)

**사람이 질문을 입력하면 → 서버가 답을 찾아서 → 채팅 화면에 답을 보여 주는 "사내 지식 챗봇"**입니다.

- 예: "건전지 어디 있어?" → "건전지는 탕비실 세 번째 서랍에 있습니다."
- 예: "탕비실에서 뭐 구할 수 있어?" → (저장된 글을 참고해서) AI가 문장을 만들어 답해 줌

---

## 2. 전체 구조 (그림으로 보기)

```
[사용자]
   │  브라우저에서 채팅 화면 열기 (localhost:5173)
   ▼
[React 앱 - client 폴더]
   │  질문 입력 → "전송" 클릭 → 서버에 POST 요청
   ▼
[Node 서버 - server 폴더]
   │  질문 받음 → ① 키워드 매칭? ② ChromaDB 검색? ③ Ollama에게 답 생성 요청?
   │
   ├── [Exact Match용 JSON 파일]  ← "건전지", "회식" 같은 키워드 → 저장된 답 그대로 반환
   ├── [ChromaDB]                 ← 비슷한 문장 검색 (의미 검색)
   └── [Ollama]                   ← 검색된 문장 + 질문을 주고 "답변 문장 만들어줘" 요청
   │
   ▼
[React 앱]
   │  서버가 준 답변을 채팅 말풍선으로 표시
   ▼
[사용자] 답을 봄
```

- **client**: 사용자가 보는 화면 (채팅 UI)
- **server**: 질문을 받아서, DB/파일/AI를 조합해 답을 만드는 곳
- **ChromaDB**: "문장"을 저장하고, "의미가 비슷한 문장"을 찾아 주는 DB
- **Ollama**: 우리 PC에서 돌아가는 AI(Llama). "이 문장들 참고해서 질문에 답해줘"라고 부르는 역할

## 4. ChromaDB가 뭔지 (쉽게)

- **일반 DB**: "건전지"라고 검색하면 "건전지"가 **딱 들어 있는** 행만 찾음.
- **ChromaDB(벡터 DB)**: 문장을 **의미 단위 숫자(벡터)**로 바꿔서 저장해 두고,  
  "탕비실에서 뭐 구할 수 있어?"처럼 **다른 말**로 물어도 **의미가 비슷한 문장**("건전지는 탕비실 세 번째 서랍에…")을 찾아줌.

**역할 정리**

- **저장**: `ingest.js`가 문장 몇 개를 ChromaDB에 넣음. (그때 **임베딩**이 필요해서 `@chroma-core/default-embed` 패키지를 씀.)
- **검색**: `index.js`에서 사용자 질문을 ChromaDB에 넘기면, "이 질문이랑 의미가 비슷한 문장 상위 N개"를 받아옴.
- **실행**: ChromaDB는 **별도 서버**로 둠. 우리는 Docker로 `chromadb/chroma` 이미지 돌려서 **8000 포트**에서 띄움. Node 서버는 이 주소로 접속해서 검색/저장함.

**한 줄:**  
ChromaDB = "문장을 의미로 검색할 수 있는 DB". 질문과 비슷한 지식 문장을 찾는 데 씀.

---

## 5. Ollama가 뭔지 (쉽게)

- **챗봇이 답을 "만드는" 건 사람이 아니라 AI.**
- **Ollama** = 우리 PC에서 돌리는 **로컬 AI 실행기**. 그 안에 **Llama** 같은 모델이 들어 있음.
- 서버(`index.js`)는 ChromaDB에서 찾은 **문장들 + 사용자 질문**을 Ollama에 넘기고,  
  "이거 참고해서 질문에 답해줘"라고 하면, Ollama가 **한국어 문장**으로 답을 만들어 줌.

**역할 정리**

- ChromaDB: **참고할 문장**을 고름.
- Ollama: 그 문장들을 **이해하고**, 질문에 맞는 **답 문장을 생성**.

**실행:**  
Ollama 앱을 켜거나, 터미널에서 `ollama run llama3` 한 번 돌려 두면, 서버가 `localhost`로 Ollama에 요청 보낼 수 있음.

**한 줄:**  
Ollama = 우리 컴퓨터에서 도는 "답 만들어 주는 AI". ChromaDB가 고른 문장을 재료로 씀.

---

## 6. 답이 나오는 순서 (플로우)

1. 사용자가 채팅 화면에서 **"건전지 어디 있어?"** 입력 후 전송.
2. **React**가 `POST /api/chat`으로 `{ question: "건전지 어디 있어?" }` 를 **Node 서버**에 보냄.
3. **Node 서버(index.js)**:
   - **① Exact Match:** `exact-match-knowledge.json`에서 "건전지" 포함된 항목 있는지 확인.  
     → 있으면: 저장된 답 + 링크 그대로 반환하고 **끝**. (ChromaDB, Ollama 안 씀.)
   - **② 없으면:** ChromaDB에 `"건전지 어디 있어?"`랑 비슷한 문장 검색.
   - **③** ChromaDB에서 문장이 **하나도 안 나오면** → "해당 정보는 등록되어 있지 않습니다. 인사/총무에 문의해 주세요." 반환.  
     이때 그 질문은 **미답변**으로 `unanswered.json`에 쌓을 수 있음.
   - **④** ChromaDB에서 **문장이 나오면** → 그 문장들 + 질문을 **Ollama**에 넘김.  
     Ollama가 "사내 지식 참고해서 답해줘" 같은 지시와 함께 **답 문장 생성**.
4. 서버가 만든 **답**을 React에 JSON으로 돌려줌. (필요하면 `refLink`도 같이.)
5. **React**가 그 답을 채팅 말풍선으로 화면에 표시.

정리하면:  
**Exact Match(키워드) → 없으면 ChromaDB(의미 검색) → 거기서 나온 걸 Ollama가 문장으로 다듬어서 답변.**

## 7. 실행할 때 뭘 켜야 하는지 (순서)

| 순서 | 뭘 켜는지 | 왜 |
|------|------------|-----|
| 1 | **ChromaDB** (Docker) | 서버가 "비슷한 문장" 검색하려면 ChromaDB 서버가 떠 있어야 함. |
| 2 | **지식 넣기** (`node ingest.js`) | ChromaDB가 비어 있으면 검색할 게 없음. 최초 1회 또는 지식 추가 후 필요 시. |
| 3 | **Node 서버** (`npm start` in server) | 채팅 요청을 받고, Exact Match / ChromaDB / Ollama 붙여서 답 만드는 주체. |
| 4 | **React 앱** (`npm run dev` in client) | 사용자가 보는 채팅 화면. 5173 포트. |
| (선택) | **Ollama** | RAG 답변(의미 검색 후 AI가 문장 만들기)을 쓰려면 필요. 안 켜면 RAG 구간은 에러나 "등록되어 있지 않습니다"만 나올 수 있음. |

**한 번에 정리:**

```text
1) Docker: chromadb (8000)
2) server: node ingest.js  (최초 1회)
3) server: npm start        (3001)
4) client: npm run dev     (5173)  → 브라우저에서 http://localhost:5173
5) (선택) Ollama 실행      (RAG 답변 쓰려면)
```

## 8. 자주 쓰는 명령어 정리

| 하고 싶은 것 | 명령어 |
|--------------|--------|

| Ollama로 대화 테스트 | `ollama run llama3` |

## 9. "배포"할 때 (나중에)

- **프론트(client)** → Vercel 같은 데 올리면, 사용자는 **URL 하나**로 채팅 화면 접속.
- **백엔드(server)** → Railway, Render 같은 데 올리면, 그 URL이 **진짜 API 주소**가 됨.
- 배포 환경에서는 보통 **ChromaDB는 Docker로 같은 서버에**, **Ollama는 안 쓰거나** 별도 서버/API로 바꾸는 식으로 구성함.  
  자세한 건 **DEPLOY.md** 참고.

## 10. 정리 (한 페이지 요약)

- **코니(Connie)** = 사내 지식 챗봇. 질문 받아서 답 찾아서 보여 줌.
- **client** = 채팅 UI (React). **server** = 답 만드는 API (Node/Express).
- **ChromaDB** = "문장을 의미로 검색"하는 DB. Docker로 8000에 띄우고, `ingest.js`로 지식 넣음.
- **Ollama** = PC에서 도는 AI. ChromaDB가 고른 문장 + 질문으로 답 문장 생성.
- **답 순서** = 키워드(Exact Match) → 없으면 ChromaDB 검색 → 그걸로 Ollama가 답 생성.
- **실행 순서** = ChromaDB → (ingest) → server → client → (선택) Ollama.

이걸 기준으로 보면 "뭐가 뭔지"가 한눈에 들어올 거예요.  
특정 파일이나 단계가 더 헷갈리면 그 부분만 따로 물어보면 됩니다.
